🎯 Working Demonstration

Demo Scenario Steps

  1. Setup: Start Node.js server and open worker dashboard
  2. Trigger: Worker presses SOS button on ESP32 device
  3. Transmission: ESP32 sends alert packet to backend API via HTTP POST
  4. Processing: Backend enriches alert with worker profile from Supabase
  5. Broadcast: Dashboard receives live alert through WebSocket
  6. Display: Alert appears on map and in bottom SOS strip
  7. Action: Control room operator clicks ACK or RESOLVE

⚙️ Technical Explanation

1. Edge Device Layer (ESP32)

Core Responsibilities

  • Read SOS button state with interrupt handling
  • Capture GPS coordinates via UART communication
  • Send HTTP POST requests to backend API
  • Maintain periodic heartbeat for device status
  • Local feedback through LED indicators
  • Configuration web server for server URL management

2. Backend Layer (Node.js + Express + WebSocket)

Core Responsibilities

  • Receive and validate alert payloads
  • Fetch worker data from Supabase emergency_profiles
  • Manage alert lifecycle (active → acknowledged → resolved)
  • Broadcast real-time updates via WebSocket
  • Track device connectivity and heartbeat
  • RESTful API for alert management

3. Data Layer (Supabase)

Database Schema

  • emergency_profiles: Worker master data (name, contact, blood group)
  • worker_sos_alerts: Incident history for analytics
  • Real-time subscriptions for live updates
  • Built-in authentication and row-level security

4. Frontend Layer (Dashboard)

UI Components

  • Interactive map with Leaflet.js for location visualization
  • Real-time SOS cards with worker details
  • Device status monitoring panel
  • Acknowledge/Resolve action buttons
  • Statistics dashboard with live counters
  • WebSocket client for instant updates

🔧 Components List

Hardware Components

  • ESP32 Development Board - Microcontroller with WiFi
  • Push Button - SOS trigger mechanism
  • GPS Module - Location tracking (NEO-6M)
  • LED/Buzzer - Local feedback indicators
  • Battery + Charging - Power management
  • Enclosure - Protective housing
  • Jumper Wires - Circuit connections

Software Components

  • Node.js Runtime - Backend JavaScript environment
  • Express Server - REST API framework
  • WebSocket (ws) - Real-time communication
  • Supabase Client - Database and authentication
  • Leaflet.js - Interactive mapping library
  • ArduinoJSON - JSON parsing on ESP32
  • TinyGPS++ - GPS data parsing
